## Quick Reference

### The Three Layers

```
┌─────────────────────────────────────────┐
│     USER SOVEREIGNTY BOUNDARY           │  ← Keys, plaintext (user controls)
├─────────────────────────────────────────┤
│     SERVICES LAYER (Optional)           │  ← Enterprise, compliance (can die)
├─────────────────────────────────────────┤
│     SOVEREIGN CORE (Protocol)           │  ← Routing, federation (unkillable)
└─────────────────────────────────────────┘
```

### The Ten Principles (Summary)

1. Non-violence is non-negotiable
2. Infrastructure, not agitation  
3. Truth without permission
4. Sovereignty at the edge
5. The protocol survives the operator
6. Layered separation protects legitimacy
7. Safe defaults, deliberate power
8. Minimal retention, maximum deletion
9. Legitimacy through transparency
10. No enemies, only problems

### Key Decisions

| Decision | Rationale |
|----------|-----------|
| Signal Protocol | Industry standard, audited, trusted |
| Federated relays | No single point of failure |
| No content on servers | Cannot be compelled to produce what we don't have |
| Enterprise revenue | Sustainable without compromising core |
| Open source protocol | Survives company, builds legitimacy |
