# Railgun Legitimacy Checklist

**Version 1.0 — January 2026**

---

## Purpose

This checklist ensures every product decision, public communication, and feature design maintains Railgun's legitimacy as civic infrastructure—not a weapon, not an ideology, not a threat.

**Use this checklist:**
- Before shipping any feature
- Before publishing any content
- Before any media appearance
- During quarterly reviews
- After any incident

---

## Section 1: Non-Violence Compliance

### 1.1 Language Review

| Check | Status |
|-------|--------|
| No military metaphors (attack, destroy, fight, battle, war) | ☐ |
| No violent imagery (weapons, explosions, combat) | ☐ |
| No revolutionary framing (overthrow, uprising, revolution) | ☐ |
| No us-vs-them framing (enemies, adversaries, opposition) | ☐ |
| No calls to action that could be misread as violent | ☐ |

**Preferred language:**

| Instead of | Use |
|------------|-----|
| "Fight censorship" | "Resilient to censorship" |
| "Defeat surveillance" | "Private by design" |
| "Weapon against tyranny" | "Infrastructure for communication" |
| "Attack vector" | "Security boundary" |
| "Kill switch" | "Shutdown mechanism" |
| "Enemies" | "Threat actors" or specific description |

### 1.2 Feature Review

| Check | Status |
|-------|--------|
| Feature cannot be used to coordinate violence | ☐ |
| Feature does not enable targeting individuals | ☐ |
| Feature does not enable mob formation | ☐ |
| Feature does not enable coercion or blackmail | ☐ |
| Feature does not celebrate or reward conflict | ☐ |

## Section 2: No Coercive Framing

| Check | Status |
|-------|--------|

| No urgency manipulation ("Act now before it's too late!") | ☐ |
| No shame-based messaging ("If you have nothing to hide...") | ☐ |

| No conspiracy framing ("They don't want you to know") | ☐ |

| Instead of | Use |

| Check | Status |
|-------|--------|

## Section 7: Quarterly Review Questions

Answer these questions every quarter:

### 7.1 Violence Assessment
1. Has Railgun been associated with any violent events? 
2. If yes, what did we learn? What changes are needed?
3. Have we received reports of the platform being used for harassment?
4. How have we responded to abuse reports?

### 7.2 Legitimacy Assessment
1. How do journalists describe us? (Collect clips)
2. How do regulators describe us? (Collect statements)
3. Have we been cited positively by civil liberties organizations?
4. Have we been cited negatively by anyone? Why?

### 7.3 Language Audit
1. Review last quarter's blog posts against this checklist
2. Review last quarter's social media against this checklist
3. Review any press releases against this checklist
4. Document any violations and corrections

### 7.4 Feature Audit
1. List all features shipped last quarter
2. Run each through Section 1.2 and 3.2 checklists
3. Document any concerns raised
4. Plan any necessary modifications

## Section 8: Incident Response

### If violence is associated with Railgun:

**Immediate (0-4 hours):**
- [ ] Executive team convenes
- [ ] Legal counsel engaged  
- [ ] NO public comment until facts established
- [ ] Internal communication to staff

**Assessment (4-24 hours):**
- [ ] Determine actual involvement (if any)
- [ ] Review what data exists (metadata only)
- [ ] Prepare factual statement
- [ ] Coordinate with law enforcement if warranted

**Response (24-72 hours):**
- [ ] Public statement emphasizing:
  - Condolences to affected
  - Charter commitment to non-violence
  - Cooperation within architectural limits
  - No celebration or defensiveness
- [ ] Internal review initiated

**Follow-up (1-4 weeks):**
- [ ] Charter review: any changes needed?
- [ ] Product review: did features enable misuse?
- [ ] Legitimacy checklist: what should change?
- [ ] Transparency report update
