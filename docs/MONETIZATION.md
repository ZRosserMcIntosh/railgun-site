# Railgun Monetization Strategy

**Version 1.0 — January 2026**

---

## Core Principle

**We sell the bridge, not the road.**

The Sovereign Core (protocol) is free, open, and unkillable. The Services Layer provides reliability, performance, compliance, and convenience—and that's where revenue comes from.

This separation ensures:
1. The protocol remains independent of commercial pressure
2. Users always have a free path (run their own relay)
3. Revenue comes from value-add, not rent-seeking on truth

| Preferential routing | Neutral infrastructure |

## Revenue Streams

### Stream 1: Consumer Premium (Low Priority, Long-term)

| Tier | Price | Features |
|------|-------|----------|
| **Free** | $0 | Full E2E messaging, groups up to 50, 1GB storage |
| **Pro** | $5/month | Groups up to 500, 50GB storage, priority support |

**Philosophy:** Free tier must be fully functional for ordinary use. Premium is for power users who want more capacity, not gated security features.

**What's NOT behind a paywall:**
- Encryption (always on, always free)
- Basic messaging (1:1, small groups)
- Security features (safety numbers, disappearing messages)
- Privacy defaults (all the same regardless of tier)

## Pricing Philosophy

### 1. Security is not a luxury
Encryption, privacy defaults, and security features are the same for free and paid users. We don't create a two-tier security system.

### 2. We compete on operations, not access
Paid tiers get better uptime, more storage, faster support—not access to content or special treatment in the protocol.

### 3. Transparent pricing
All prices are public. No "contact sales" for basic tiers. Enterprise custom pricing is for genuinely custom needs, not price discrimination.

### 4. No lock-in
Data export is available to all users. Protocol is open. Switching costs are low by design.

## Revenue Ethics

### Questions we ask before any revenue decision:

1. **Does this compromise the protocol?** If yes, don't do it.
2. **Does this create security inequality?** If yes, reconsider.
3. **Does this lock users in?** If yes, add export options.
4. **Would we be comfortable if this pricing was public?** If no, change it.
5. **Does this align with the Charter?** If unclear, get ethics review.

### Revenue we will refuse:

- Payment for backdoors (any amount)
- Payment for user data (any amount)
- Payment for preferential protocol treatment
- Government contracts requiring compromise
- Advertising revenue
